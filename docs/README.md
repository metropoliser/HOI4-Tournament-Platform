## Contributing to Documentation

When contributing to documentation:

1. Keep user-facing docs in the root directory
2. Keep developer/technical docs in this `docs/` folder
3. Update this README when adding new documentation
4. Use clear headers and formatting
5. Include code examples where appropriate
6. Keep documentation up-to-date with code changes

## Documentation Standards

- Use GitHub-flavored Markdown
- Include a table of contents for long documents
- Use code blocks with language syntax highlighting
- Include links to related documentation
- Keep explanations clear and concise
- Update the main README when adding new docs
