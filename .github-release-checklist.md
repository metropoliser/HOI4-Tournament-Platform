# GitHub Release Checklist

Before pushing to GitHub, complete these steps:

## Pre-Release Checklist

- [ ] All sensitive files are in .gitignore (.env.local, .env.docker, etc.)
- [ ] Remove or clean up any temporary files
- [ ] Update package.json with correct repository URL
- [ ] Update README.md with correct GitHub repository links
- [ ] Update CHANGELOG.md with correct repository links
- [ ] Review all code for any hardcoded secrets or credentials
- [ ] Test the installation instructions in README.md

## Initialize Git Repository

```bash
# Initialize git if not already done
git init

# Add all files
git add .

# Create initial commit
git commit -m "Initial commit: HOI4 Tournament Platform v1.0.0"

# Create main branch (if needed)
git branch -M main

# Add remote repository
git remote add origin https://github.com/YOUR_USERNAME/hoi4-tournament-platform.git

# Push to GitHub
git push -u origin main
```

## Create First Release

1. Go to your GitHub repository
2. Click on "Releases" in the right sidebar
3. Click "Create a new release"
4. Tag version: `v1.0.0`
5. Release title: `v1.0.0 - Initial Release`
6. Description: Copy content from CHANGELOG.md [1.0.0] section
7. Click "Publish release"
